MISSION 01 · INITIAL SETUP
Install, login, configure LHOST/LPORT, verify the server is reachable
STEP 1 · START THE SERVER
What to do: From the repo root, run uv sync then uv run python server.py (add --ssl for HTTPS).
Expected result: Terminal shows Uvicorn listening on port 1337.
Pass if: Browser opens http://127.0.0.1:1337 and the login screen appears.
Troubleshooting: Port in use - stop other TSK instances or run fuser -k 1337/tcp on Linux.
STEP 2 · REGISTER & LOGIN
What to do: Click NEW, pick an operator name, set a password (8+ chars), register, then log in.
Expected result: Main UI loads with sidebar devices and payload list.
Pass if: Header shows your operator name and PAYLOADS count.
STEP 3 · SET LHOST & LPORT
What to do: Open CONFIG (sidebar or C). Click the LAN IP helper next to LHOST, or type your machine's LAN IP manually. Set LPORT to 1337 unless you changed the server port.
Expected result: Bottom bar and header show your LAN IP in green instead of NOT SET.
Pass if: set lhost in the command bar reflects the saved value after reload.
Troubleshooting: Phone-home will fail if LHOST is 127.0.0.1 and the target is another machine on the LAN - use your real LAN IP.
STEP 4 · CLONE A PAYLOAD REPO (OPTIONAL)
What to do: Open PAYLOAD REPOS, click CLONE on Rubber Ducky (or any device you plan to use).
Expected result: Toast confirms clone; payload counts update in the sidebar badges.
Pass if: BROWSE tab lists payloads for that device.
MISSION 02 · USB EXFIL - BUILD & FLASH
SnarfSnarf builder, format options, save, and flash to a plain USB stick
Use a blank FAT32 or exFAT USB stick - not a Bash Bunny volume, not a Windows install ISO.
STEP 1 · OPEN SNARF BUILDER
What to do: Sidebar → USB DROPPER. Center tab → SNARF. Confirm EXFIL sub-tab is active.
Expected result: Exfil targets checklist, output/format dropdowns, and BUILD PREVIEW button visible.
Pass if: Right preview panel shows generated script after clicking BUILD PREVIEW.
STEP 2 · CONFIGURE TARGETS & OUTPUT
What to do:
  • Check exfil targets (e.g. Documents, Credentials, Clipboard).
  • Pick Phone home - HTTP POST for CATCH tab testing, or Copy to USB root for offline exfil.
  • Choose PowerShell [Windows] or Bash [Linux] format.
  • Set filename (default snarf_exfil).
Expected result: Preview script includes your LHOST/LPORT when phone-home is selected.
Pass if: Preview URL contains http://<your-lhost>:1337/api/snarf (or your LPORT).
Troubleshooting: Missing LHOST in script - return to Mission 1 Step 3.
STEP 3 · SAVE TO PAYLOAD LIBRARY
What to do: Click SAVE TO PAYLOADS (or SAVE + FLASH if stick is already plugged in).
Expected result: Toast confirms save. Switch to BROWSE tab - script appears under SNARFSNARF category.
Pass if: Selecting the saved payload shows its content in the right preview panel.
STEP 4 · PLUG USB & FLASH
What to do:
  • Plug a writable USB stick. Wait for USB DRIVE status → READY in sidebar.
  • Right panel: confirm stick is detected. Pick deploy target (USB root or hidden .tsk/).
  • Select your saved SNARF payload in BROWSE, then click FLASH (or f).
Expected result: Flash log shows copy progress. Stick contains script + RUN_PAYLOAD.bat (Windows) or RUN_PAYLOAD.sh (Linux).
Pass if: Toast says flash OK and terminal log shows the deploy path.
Troubleshooting: read-only error - reformat stick as FAT32/exFAT. No stick detected - try another port; check devices command.
STEP 5 · TRIGGER ON TARGET (MANUAL TEST)
What to do: On a lab Windows VM or Linux box, open the stick and run the launcher (or execute the script directly).
Expected result: Script runs without errors. Phone-home mode posts files to TSK; USB mode copies files to the stick.
Pass if: Target can reach http://<LHOST>:<LPORT>/api/snarf over the LAN (Mission 3 verifies catches).
MISSION 03 · USB EXFIL - CATCH & REVIEW
Watch phone-home uploads land in the CATCH tab, preview files, export
STEP 1 · OPEN CATCH TAB
What to do: USB DROPPER → SNARF tab → click CATCH sub-tab.
Expected result: LIVE FEED and SNARFED FILES panels visible. Feed may say "Waiting for phone-home uploads..."
Pass if: CATCH tab loads without errors while logged in as the operator who built the script.
STEP 2 · RUN PHONE-HOME EXFIL ON TARGET
What to do: Execute the flashed script from Mission 2 on a lab machine that can reach your LHOST over the network.
Expected result: LIVE FEED shows new lines with hostname, timestamp, and filepath as uploads arrive.
Pass if: A new session folder appears under SNARFED FILES grouped by hostname.
Troubleshooting:
  • No feed activity - ping LHOST from target; check firewall on TSK machine (allow inbound TCP on LPORT).
  • Wrong operator's catches - each operator has isolated storage; log in as the builder.
  • HTTP blocked - try same-subnet test first before crossing VLANs.
STEP 3 · PREVIEW & DOWNLOAD
What to do: Click a snarfed file in the list. Use preview in the right panel. Click download on individual files or EXPORT ALL for a ZIP.
Expected result: File contents render in preview (text) or download prompt (binary).
Pass if: Downloaded file matches what the target exfiltrated (e.g. a test document you planted).
STEP 4 · REFRESH & VERIFY ISOLATION
What to do: Click REFRESH. Optionally log in as a second operator and confirm their CATCH tab does not show the first operator's files.
Expected result: Catches persist after refresh. Second operator sees empty feed (unless they ran their own script).
Pass if: Files exist on disk at snarfed/<operator>/ on the TSK server.
MISSION 04 · LURE PACKAGE
Build LNK/README lure, attach exfil script, flash the full package
STEP 1 · OPEN LURE BUILDER
What to do: USB DROPPER → SNARF → LURE sub-tab.
Expected result: Lure type selector (Windows LNK / Linux README / .desktop), package name field, payload attachment dropdown.
Pass if: Saved exfil scripts from Mission 2 appear in the attach dropdown.
STEP 2 · BUILD & SAVE PACKAGE
What to do: Pick lure type, name the package, attach your exfil script (or a test stub). Click BUILD PREVIEW, then SAVE PACKAGE.
Expected result: Package saved under BROWSE → LURES category with multiple files in the manifest.
Pass if: Preview lists lure file + payload script + launcher files together.
STEP 3 · FLASH PACKAGE TO STICK
What to do: Select the lure package in BROWSE. Click FLASH PACKAGE (or FLASH - TSK detects lure manifests automatically).
Expected result: All package files copy to the stick. Windows stick shows a convincing shortcut or README; script stays alongside.
Pass if: Target user double-clicking the lure runs the attached exfil (lab test only).
Troubleshooting: LNK blocked by policy - test README lure on Linux or use manual script execution for lab validation.
MISSION 05 · BASH BUNNY
Mount detection, payload select, switch position, ARMED state, flash
STEP 1 · CONNECT BUNNY
What to do: Plug Bash Bunny in arming mode (switch toward USB logo). Set BUNNY MOUNT in CONFIG if auto-detect fails.
Expected result: Sidebar BASH BUNNY shows READY. ARMED badge may show STANDBY until switch position is valid.
Pass if: devices command lists bunny as connected with a mount path.
STEP 2 · SELECT PAYLOAD & SWITCH
What to do: Navigate to BASH BUNNY. Clone repo if needed. Pick a payload from BROWSE. Set switch position 1 or 2 in the header.
Expected result: Preview shows payload.txt content. Switch buttons highlight active position.
Pass if: flash 1 or flash 2 in command bar targets the selected switch.
STEP 3 · FLASH & DEPLOY
What to do: Click FLASH. Flip Bunny switch to payload mode (away from USB logo) and plug into target.
Expected result: Flash log shows copy to payloads/switchN/ on the Bunny mass-storage volume.
Pass if: Bunny executes payload on target (LED pattern / expected target behavior per payload README).
Troubleshooting: ARMED stays STANDBY - Bunny not in arming mode or mount path wrong. Re-seat switch and refresh CONFIG mount.
MISSION 06 · LAN TURTLE
SSH connect, browse modules, flash over the network
STEP 1 · CONFIGURE TURTLE SSH
What to do: CONFIG tab → set TURTLE IP (default 172.16.84.1), user root, port 22, password if set.
Expected result: Turtle row in sidebar shows READY when reachable on the network.
Pass if: turtle connect opens the in-browser SSH terminal.
STEP 2 · BROWSE & PREVIEW MODULE
What to do: Sidebar → LAN TURTLE. Clone turtle modules repo. Select a module payload from BROWSE.
Expected result: Preview shows module scripts. Source indicator shows repo path.
Pass if: Payload list populates after clone completes.
STEP 3 · FLASH OVER SSH
What to do: With Turtle connected, select payload and click FLASH. TSK copies module files over SSH/SFTP.
Expected result: Flash log shows remote copy progress. Module appears on Turtle filesystem.
Pass if: SSH terminal shows deployed files under the expected Turtle path.
Troubleshooting: Auth failure - try blank password (press Enter) or default hak5lan. Host key errors are handled automatically.
MISSION 07 · RUBBER DUCKY
SD card prep, payload select, encode, manual flash
STEP 1 · PREP DUCKY SD CARD
What to do: Insert microSD in Rubber Ducky (or SD adapter). Plug Ducky into TSK machine. Set DUCKY MOUNT in CONFIG if not auto-detected.
Expected result: Sidebar RUBBER DUCKY shows READY when the volume mounts.
Pass if: CONFIG shows a valid mount path for the Ducky volume.
STEP 2 · CLONE & SELECT PAYLOAD
What to do: Clone the Ducky payload repo. Browse payloads, pick one (start with a harmless EXEC or RECON example).
Expected result: Preview shows Ducky Script (REM, STRING, ENTER commands).
Pass if: Payload preview renders with syntax highlighting.
STEP 3 · SET ENCODER & FLASH
What to do: CONFIG → set path to encoder.jar (Hak5 Duck Encoder). Select payload, click FLASH.
Expected result: TSK encodes script to inject.bin and writes it to the Ducky root.
Pass if: Flash log reports encode + copy success; inject.bin exists on the SD volume.
Troubleshooting: Encoder not found - install Java, download encoder.jar, set path in CONFIG. Mount missing - re-seat SD and refresh devices.
STEP 4 · DEPLOY ON TARGET
What to do: Eject safely, insert Ducky in target machine USB port.
Expected result: Ducky runs inject.bin - keystrokes inject per payload (lab VM recommended).
Pass if: Target shows expected payload behavior within seconds of plug-in.
